# Suggested Reading

## High-Priority Reading

### Designing Data-Intensive Applications - Martin Kleppmann

Why it matters:

- strongest single book for system design under scale, consistency, messaging, storage, and distributed tradeoffs

How it improves the system:

- would deepen distributed systems reasoning, data ownership, event flows, and reliability guidance across architecture and database skills

### Site Reliability Engineering - Google

Why it matters:

- foundational for service reliability, SLOs, toil reduction, and production operations

How it improves the system:

- would help create missing skills for observability, reliability engineering, alerting, and operational maturity

### Software Engineering At Google

Why it matters:

- strong on maintainability, scale, testing, code health, and engineering process

How it improves the system:

- would strengthen repository-wide guidance on code review, testing strategy, dependency management, and long-term maintainability

### Release It! - Michael Nygard

Why it matters:

- practical patterns for stability, resilience, bulkheads, circuit breakers, and failure containment

How it improves the system:

- would deepen the repository's failure-mode and production-hardening guidance

### Building Secure And Reliable Systems - Google

Why it matters:

- integrates security and reliability rather than treating them separately

How it improves the system:

- would help the repository produce stronger cross-domain security-plus-operability guidance

## Architecture And Domain Design

### Domain-Driven Design - Eric Evans

Why it matters:

- canonical reference for bounded contexts, domain language, and strategic design

How it improves the system:

- would sharpen architecture and ERP-oriented domain modeling skills

### Implementing Domain-Driven Design - Vaughn Vernon

Why it matters:

- more implementation-focused than Evans and better suited to turning domain design into reusable patterns

How it improves the system:

- would improve modular SaaS, ERP workflow design, and architecture skill handoffs

### Fundamentals Of Software Architecture - Richards and Ford

Why it matters:

- practical coverage of architecture characteristics and tradeoff analysis

How it improves the system:

- would strengthen the repository's architecture decision and tradeoff depth

## Testing And Delivery

### Accelerate - Forsgren, Humble, Kim

Why it matters:

- strong evidence-based treatment of delivery performance and engineering effectiveness

How it improves the system:

- would improve release engineering, deployment workflow, and delivery metrics guidance

### Continuous Delivery - Humble and Farley

Why it matters:

- definitive reference for safe, repeatable release pipelines

How it improves the system:

- would support stronger CI/CD, deployment safety, and release verification skills

## Security And AI

### The Pragmatic Programmer

Why it matters:

- still one of the best books for engineering habits, code quality, and risk-aware implementation

How it improves the system:

- would strengthen repo-wide engineering judgment and code craftsmanship guidance

### Secure By Design - Dan Bergh Johnsson, Daniel Deogun, Daniel Sawano

Why it matters:

- focuses on security built into design decisions rather than bolted on later

How it improves the system:

- would deepen baseline security-by-design guidance and improve integration between architecture and security skills

### OWASP ASVS

Why it matters:

- practical application security verification standard

How it improves the system:

- would give the repository a stronger shared verification model for production readiness

## Standards And Playbooks

### OpenTelemetry Specifications

Why it matters:

- current industry standard for traces, metrics, and logs

How it improves the system:

- would inform a first-class observability skill with realistic implementation guidance

### Google API Design Guide

Why it matters:

- strong consistency standard for contract design

How it improves the system:

- would improve API surface quality, naming consistency, and long-term evolution safety

### AWS Well-Architected Framework

Why it matters:

- broad but useful for operational excellence, security, reliability, performance efficiency, and cost

How it improves the system:

- would help the repository create a stronger holistic system-review discipline
