# Delivery Management Scorecard

Use this reference to review whether the engineering system is improving or degrading.

## Core Metrics

- deployment frequency
- lead time for changes
- change failure rate
- mean time to recover
- flaky test rate or quarantined test count
- pipeline red-time or broken-build duration
- incident volume and repeat-incident rate

## Review Prompts

- What slowed delivery this period?
- Which quality problem most reduced trust in the pipeline or release process?
- Which repeated manual task should be automated next?
- Which bottleneck is structural rather than individual?
