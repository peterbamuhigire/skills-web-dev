# Natoli Enterprise Playbook

Use this reference to run fast, evidence-led discovery for enterprise product teams.

## Core rules

1. Stop jumping to solutions.
2. Get the right people in the room early.
3. Use fast user context instead of heavy persona theater.
4. Prototype to reveal requirements.
5. Distinguish task completion from actual success.
6. Table or drop features that stay undefined.

## One-page discovery brief

Capture:

- Problem statement
- Target user and situation
- Current workaround
- Constraint or dependency
- Definition of success
- Known unknowns

If the team cannot write this plainly, it is not ready for delivery planning.

## Requirement rule

Do not treat user stories as the first discovery artifact. First create:

1. Contextual scenario
2. Action flow
3. Low-fidelity prototype
4. Validation notes

Then write requirements that the prototype made visible.

## Stakeholder rule

Calm fear with clarity:

- What user problem are we solving?
- What are we not deciding yet?
- What evidence will settle the disagreement?

This reduces political churn better than more slideware.
