# Excel Quality Checklist

Run this checklist before delivering any .xlsx file to a user, client, or system.

---

## Structure & Data Integrity

- [ ] All data ranges converted to Excel Tables (`Ctrl+T`) with descriptive names
- [ ] No unrelated datasets mixed on the same sheet
- [ ] No merged cells in data rows (presentation headers only)
- [ ] No blank rows or columns within a data table
- [ ] All tables have unique, non-blank column headers in row 1
- [ ] No stray data outside Table definitions

## Number Formatting

- [ ] No cells in a Number/Currency/Date column left as "General" format
- [ ] Currency columns use `#,##0` or `#,##0.00` — not raw numbers without thousands separator
- [ ] Date columns use a consistent, explicit date format (`DD MMM YYYY` or `YYYY-MM-DD`)
- [ ] Percentage columns use `0%` or `0.00%` — not raw decimal values (0.12 instead of 12%)
- [ ] All currency in the document uses the same currency symbol and format
- [ ] Large numbers (millions) use appropriate abbreviation format where applicable

## Design & Layout

- [ ] Title row present at the top of each sheet (branded, formatted, correct content)
- [ ] Table header row is visually distinct (brand colour fill, white bold text)
- [ ] Column widths set appropriately — no truncated content, no excessive whitespace
- [ ] Row heights set: title 30–36pt, header 22–24pt, data rows 18pt minimum
- [ ] Text columns left-aligned, numeric/date columns right-aligned
- [ ] Colour palette limited to 3 brand colours + greys
- [ ] No 3D chart effects, no gradient cell fills, no decorative clip art

## Navigation & Usability

- [ ] Header row frozen (freeze panes set at the row below the table header)
- [ ] Wide tables: first ID/Name column also frozen
- [ ] Sheet tabs named clearly — no "Sheet1", "Sheet2", "Sheet3"
- [ ] Tab colours assigned in multi-sheet workbooks
- [ ] Active sheet on open is Summary or first data sheet
- [ ] AutoFilter arrows present on all Table columns

## Formulas

- [ ] No #REF!, #VALUE!, #NAME?, or #DIV/0! errors visible in any cell (use IFERROR where intentional)
- [ ] All lookups use XLOOKUP (not VLOOKUP) for new formulas
- [ ] Structured Table references used (`Table[Column]`) not raw cell addresses in formulas
- [ ] No hard-coded values inside formulas — constants are in named cells or Config sheet
- [ ] Complex formulas use LET for readability

## Data Validation

- [ ] All user-input columns have data validation applied
- [ ] Dropdown lists include input message and Stop-type error alert
- [ ] Date fields restrict to valid date ranges

## Conditional Formatting

- [ ] Status columns use traffic-light conditional formatting (green/amber/red)
- [ ] Rules applied to full Table column references, not fixed cell ranges (so they auto-expand)
- [ ] No conflicting conditional formatting rules on the same range

## Charts (if present)

- [ ] No 3D charts
- [ ] All charts have informative titles (not "Chart 1")
- [ ] Chart colour palette uses brand colours in sequence
- [ ] No chart borders; white/transparent backgrounds
- [ ] Charts sized consistently and snapped to cell grid
- [ ] Axes labelled with units

## Print Setup (for sheets intended for printing)

- [ ] Orientation set (landscape for wide tables)
- [ ] Scale: Fit to 1 page wide
- [ ] Print titles set (header rows repeat on every page)
- [ ] Page header and footer configured
- [ ] Print preview checked — no orphaned columns on a second page

## Workbook Hygiene

- [ ] No unused default sheets remaining
- [ ] Document properties set: Title, Author, Company
- [ ] Calculation set to Automatic
- [ ] PivotTables refreshed (if any)
- [ ] File saved as .xlsx (not .xls or .xlsm unless macros required)
- [ ] File size is reasonable — no embedded images at excessive resolution
- [ ] Sensitive data removed or protected if distributing externally

## Programmatic Output (if generated by code)

- [ ] Table definition added (`add_table` / `addTableStyleInfo`) — not just formatted rows
- [ ] Number formats set on all numeric columns in code
- [ ] Column widths set in code (auto-calculated from content)
- [ ] Header row freeze applied in code
- [ ] Sheet tab colour set in code
- [ ] File tested by opening in Excel — not just verified programmatically
