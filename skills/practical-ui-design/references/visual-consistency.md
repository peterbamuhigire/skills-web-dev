# Visual Consistency

Use this reference when designing or auditing whether the interface feels coherent,
predictable, and easy to read.

## Consistency is not sameness

Keep the logic stable:

- same control types for the same actions
- same spacing rhythm for the same hierarchy
- same labeling pattern for the same concepts
- same state treatment for the same interaction class

Do not flatten the brand into identical screens. Preserve identity while keeping the
system legible.

## The five surfaces to check

1. Layout
2. Typography
3. Color and emphasis
4. Component shape and state
5. Language and labels

Weakness in any one of these can make the product feel inconsistent even if the others
are polished.

## Signifier rule

Interactive elements should look interactive without hover. If a user has to guess what
is clickable, the visual system is failing.

## Expectation rule

Check category norms before inventing a new pattern. Novelty is allowed only when the
benefit is obvious and learnable.
