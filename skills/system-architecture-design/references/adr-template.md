# ADR Template

Use this format for architecture decisions that affect modules, contracts, data ownership, or operations.

## Decision

- **Title**
- **Status**: proposed, accepted, superseded, rejected
- **Date**
- **Owners**

## Context

- What problem are we solving?
- What constraints matter?
- What assumptions are we making?
- What would fail if we did nothing?

## Options

For each option include:

- Shape
- Benefits
- Risks
- Operational cost
- Migration cost

## Decision

State the selected option and why it wins now.

## Consequences

- What becomes easier?
- What becomes harder?
- What follow-up work is required?
- What signals would tell us to revisit this decision?
